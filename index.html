    The Developer Portal

    What is the Developer Portal?

    If you are building a great app or a new product/feature based on Revolut Business, then the Developer Portal is for you.
    The Developer Portal allows you to easily create, manage and distribute apps.
    Apps allow you to ask Revolut users to give you permission to use their account on their behalf. Once you will feel confident enough in your integration, the Developer Portal will allow you to submit and publish your apps and integrations.

    Signin Up

    • To get an access to Revolut Open Banking API you need to signup to Revolut Developers Portal. After you sign up, please set your Open Banking Organisation Id in Profile section. This will help us to link your applications with your developer account during dynamic application registration process

    • Once you created your account and set your organization id, please get in touch with business-partners@revolut.com and ask for a tet sandbox user credentials. We will provide you with a demo Revolut user account credentials (phone number, password).

    How do developers portal work?

    Open Banking API ,for our developer portal, is an extension for our existing Open Business APIs project. In the portal you can create Open Business APIs as usual (for more information https://revolutdev.github.io/business-api/ ) and you can view or manage your Open Banking API application. Open Banking Applications can only be created by dynamic client registration as it is described in the offical documentation.

    Open Id Configuration URL

    • Production: https://oba.revolut.codes/openid-configuration
    • Sandbox: https://sandbox-oba.revolut.codes/openid-configuration

    Dynamic Client Registration

    All API endpoints are implemented as it is described in the official documentation

    curl -X POST https://sandbox-oba.revolut.com/dynamic-client-registration/register

    curl -X POST https://oba.revolut.com/dynamic-client-registration/register

    This endpoint will work only if you register to developer portal and set open banking organisation id from the portal for your account.

    Open Banking Security Profile

    Open Banking security profile is implemented as it is desribed in the official documentation

    Authorization Endpoint

    curl -X POST https://sandbox-oba.revolut.com/ui/index.html

    curl -X POST https://oba.revolut.com/ui/index.html

    /ui/index.html endpoint is for your application to redirect Revolut customer to authorize consent

    Token Endpoint

    curl -X GET https://sandbox-oba-auth.revolut.com/token

    curl -X GET https://oba-auth.revolut.com/token

    /token endpoint is beign served from Revolut authorization server. These endpoints requires client certificate authentication for MATLS

    Account and Transaction API

    Open Banking Account and Transaction API is implemented as specified in the official documentation

    The Interfaces already implemented can be seen below. By the time, we will be implementing other endpoints (manadatory endpoints first). We will be updating this documentation.

    Account Access Consent

    Cancel/Revoke Consent

    curl -X DELETE curl https://sandbox-oba.revolut.com/account-access-consents/{consent-id}

    curl -X DELETE curl https://oba.revolut.com/account-access-consents/{consent-id}

    This end point:

    • Rejects consent if a consent in AwaitingAuthorisation state

    • Revokes consent if consent in Authorised state

    Get Consent

    curl -X GET curl https://sandbox-oba.revolut.com/account-access-consents/{consent-id}

    curl -X GET curl https://oba.revolut.com/account-access-consents/{consent-id}

    This endpoint returns consent details as it is described in the official documentation

    Errors

    The API uses the following error codes:

    Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- Access to the requested resource or action is forbidden.
    404 Not Found -- The requested resource could not be found.
    405 Method Not Allowed -- You tried to access an endpoint with an invalid method.
    406 Not Acceptable -- You requested a format that isn't JSON.
    429 Too Many Requests -- You're sending too many requests.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
